In blue color, we designate the instructions related to the old decentralized data storage, which is currently active by default.

OptiChar stores databases with user data on the hard disk. Data related to specific problems can be organized into separate directories, known as Problem directories. This decentralized storage is particularly useful when multiple design or characterization problems are being analyzed simultaneously or when the software is used by multiple users.

The File Transfer option allows users to transfer data files between different Problem directories, reducing preparation time. For example, substrate data files for various types of glass can be easily transferred from the Catalog directory.

The Backup/Restore option is another convenient feature in OptiChar. It lets users create a backup file (*.zip) of any Problem directory, which can later be restored on this or any other computer with OptiChar installed. This feature provides a straightforward method for data exchange between users and systems.

Before starting any analysis or synthesis procedure, all necessary data must be loaded into memory. This includes substrate information, measurement data, and optionally, layer material properties. Data files containing this information are stored in appropriately named databases accessible from the Data menu.

The commands that access databases in the Data menu are: Measurements Data, Substrate, Layer, and Layer Model.


Groups of database files:

Measurements Data

Measurements data  files store a collection of measured values for various characteristics, which can vary depending on wavelength or angle of incidence. This data is archived in the Measurements database and can be accessed using the Measurements Data command in the Data menu. The commands that access these databases in the Data menu include: - Measurements Data - Substrate - Layer - Layer Model

A measurements data file may include values for multiple characteristics, such as: - Reflectance - Transmittance - Ellipsometric angles (Psi and Delta) - Group Delay (GD) - Group Delay Dispersion (GDD) These values can be recorded for one or more polarization states (when applicable) and across multiple angles of incidence.

Materials Files

The Layer Materials command opens the Layer Materials database.   Users can either transfer a layer material file from an existing database within a different problem menu or generate a new file.  Layer materials are characterized by their refractive indices, which vary as a function of wavelength:   - Non-dispersive materials: Specified by a single refractive index value.   - Dispersive materials: Specified in tabular format, detailing the real and imaginary parts of the refractive index at various wavelengths.

Users can specify substrate, incident medium, and layer materials using spreadsheet formats for dispersive data or through refractive index formulas, which include:   - Sellmeier formulas   - Cauchy formulas   - Hartmann and Hartmann-2 formulas   - Schott Glass formula   - Drude formula   - Spline interpolation

Substrate

The Substrate command opens the Substrate database.  A substrate material is specified similarly to a layer material. Files from the Substrate database can be used as:   - Substrates   - Incident Medium   - Exit Medium

  Users can utilize the right-click popup menu in the Database window for this purpose.

Layer Model

Layer Model database stores models obtained by OptiChar during characterization procedure.  The Layer Model database is accessed using the Layer Model command in the Data menu. These models can be loaded to memory at any time and used, for example, Users can load these models into memory at any time, using them as starting approximations for further characterization activities. The Layer Model database features a specialized Layer Model Editor that allows users to adjust significant parameters of the model as necessary.
