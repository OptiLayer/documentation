Layer Material Abbreviations

` <idh_design.html>`__ ` <idh_design.html>`__ ` <idh_design_editor.html>`__

OptiLayer design files use two-character abbreviations to identify layer materials instead of explicitly naming them. This approach simplifies the general description of the design. Upon loading materials from a database, default abbreviations are automatically assigned. The first two materials, based on their refractive index at the reference wavelength specified in the Configuration Options, are labeled “L” for the material with the lower refractive index and “H” for the one with the higher refractive index. For dispersive materials, this process involves comparing their refractive indices at the specified reference wavelength. Subsequent materials are designated with letters in alphabetical order (A, B, C, etc.). These materials and their abbreviations are then listed in the General Information Window for quick reference.

Users have the option to customize these abbreviations via the Options command found in both the Analysis and Synthesis Menus, or through the Arrange Materials… command located in the Data menu. Additionally it is possible to specify Vacuum materials, which finds use in various tools such as the Monitoring spreadsheet, BBM and Monochromatic simulators, and for the Preproduction estimation of errors..

When abbreviations are modified, the changes are reflected across all design files currently loaded into memory. This ensures that changing an abbreviation won’t disrupt the link between design layers and their designated materials.

Note: In the current version, users can employ long abbreviations without restrictions on string length. However, it is highly recommended to restrict oneself to 1 or 2-character abbreviations, as they help maintain a simple design representation.

Long abbreviations can be more convenient than the previously utilized one-character and two-character abbreviations, especially when many materials are loaded into memory or when there is a preference to denote material names directly.
